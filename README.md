# 📊 Telegram Scraper & AI NLP Analyzer

Мощный комбайн для парсинга, управления и интеллектуального анализа данных из Telegram-каналов. Проект состоит из удобного веб-интерфейса для скачивания постов, медиафайлов и комментариев, а также встроенного ИИ-движка для классификации тем и анализа тональности с использованием локальных LLM (Ollama) и HuggingFace Transformers.

---

## ✨ Основные возможности

* **Полный парсинг каналов:** Скачивание постов, комментариев и медиафайлов (включая скрытые) через Telegram API.
* **Удобный Web-интерфейс:** Управление задачами парсинга, просмотр статистики и графиков прямо в браузере.
* **Классификация тем постов:** Автоматическое разделение постов по категориям (Политика, IT, Спорт, Юмор и т.д.) с помощью ИИ.
* **Контекстный анализ комментариев:** Определение реального отношения комментатора к автору или тексту поста (позитивное, негативное, нейтральное).
* **Гибридный NLP-движок:** Поддержка локального сервера Ollama (`qwen2.5:3b`) и моделей HuggingFace (`rubert-base`).
* **Максимальная скорость:** Использование многопоточности для Ollama и батчинга для PyTorch/HuggingFace.

---

## 🛠 Технологии

* **Frontend:** React, Tailwind CSS, Recharts, Vite
* **Backend (Scraper):** Node.js, Express, better-sqlite3, GramJS (Telegram API)
* **AI/NLP Engine:** Python 3.9+, PyTorch, Transformers, Ollama API, sqlite3

## 🚀 Установка и запуск

Проект разделен на две части: **Веб-приложение (парсер)** и **Python-скрипты (ИИ-анализатор)**. Они работают с одной и той же базой данных SQLite.

### Часть 1. Запуск Веб-интерфейса и Парсера (Node.js)

Для работы требуется [Node.js](https://nodejs.org/) (версия 18 или выше).

1. Распакуйте архив или склонируйте репозиторий в удобную папку.
2. Откройте терминал в папке проекта и установите зависимости:
   ```bash
   npm install
   ```
3. Запустите проект в режиме разработчика:
   ```bash
   npm run dev
   ```
4. Откройте браузер и перейдите по адресу, который появится в терминале (обычно это `http://localhost:3000`).

#### ⚙️ Настройка MTProto (Опционально)
Для скачивания скрытых медиафайлов и работы без жестких лимитов Telegram:
1. Получите `API ID` и `API Hash` на сайте [my.telegram.org](https://my.telegram.org/auth).
2. В интерфейсе приложения перейдите в раздел парсинга и нажмите **"Настроить MTProto"**.
3. Введите ваши данные, номер телефона и код подтверждения из Telegram.

---

### Часть 2. Запуск ИИ-анализа данных (Python)

После того как парсер скачал данные в базу (например, `telegram_export.db` или `bIoodsiker_71p_4904c.db`), можно запускать нейросети для их анализа.

1. **Создайте и активируйте виртуальное окружение (опционально):**
   ```bash
   python -m venv venv
   # Для Windows:
   .\venv\Scripts\activate
   # Для Linux/macOS:
   source venv/bin/activate
   ```

2. **Установите зависимости:**
   ```bash
   pip install tqdm transformers torch
   ```

3. **Запуск анализа тем (посты):**
   ```bash
   python analyze_topics_mode.py имя_твоей_базы.db
   ```

4. **Запуск анализа тональности (комментарии):**
   ```bash
   python analyze_comments_mode.py имя_твоей_базы.db
   ```
   *При запуске скрипт предложит выбрать режим: `1` для HuggingFace (`rubert`) или `2` для Ollama (`qwen2.5`).*

## 🧠 Настройка Ollama и загрузка Qwen

Для качественного контекстного анализа рекомендуется использовать модель **Qwen 2.5 (3B)** через Ollama. Она отлично оптимизирована под русский язык.

1. Скачайте и установите Ollama с официального сайта [ollama.com](https://ollama.com).
2. Запустите терминал и скачайте модель командой:
   ```bash
   ollama pull qwen2.5:3b
   ```

## ⚡ Настройка многопоточности в Ollama (Ускорение в разы)

### Зачем это нужно?
По умолчанию Ollama обрабатывает запросы строго по очереди (в один поток). Наш Python-скрипт умеет отправлять запросы **пачками (параллельно)**. Чтобы Ollama не ставила их в очередь, а щелкала одновременно, нужно включить параллельность на стороне самого сервера Олламы.

### Инструкция для Windows:
1. Нажмите **Win + R**, введите `sysdm.cpl` и нажмите **Enter**.
2. Перейдите во вкладку **Дополнительно** (Advanced) -> **Переменные среды...** (Environment Variables).
3. В блоке **«Переменные среды пользователя»** нажмите **Создать...** и введите:
   * **Имя переменной:** `OLLAMA_NUM_PARALLEL`
   * **Значение переменной:** `4` *(или `8`, если у вас мощный ПК с хорошей видеокартой)*
4. Нажмите **OK** во всех окнах.
5. **⚠️ ВАЖНО:** Перезапустите Ollama! Найдите её иконку в системном трее (возле часов) -> клик правой кнопкой мыши -> **Quit Ollama**, а затем запустите её заново через пуск.

## 💾 Структура базы данных (SQLite)

Обе части приложения работают с единым файлом базы данных. После работы ИИ-скриптов в вашей базе создаются две таблицы с индексами для быстрого чтения веб-интерфейсом:

* **`PostTopics`** — результаты классификации тем постов (`post_id`, `topic`, `confidence_score`).
* **`CommentRelation`** — результаты контекстного анализа комментариев (`comment_id`, `post_id`, `relation`, `confidence_score`).
